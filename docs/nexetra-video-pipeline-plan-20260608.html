Nexetra Local AI Video Production Factory

📅 Created: 2026-06-08 🗂 Workspace: nexetra-media 🖥 Lab: Mac Mini M4 ×6 + DGX Spark + ubuntu-1

Goal

Multi-lingual promotional video pipeline running 90%+ locally on the Mac Mini lab + DGX Spark. One trigger → script → translated voices → visuals → assembled multi-format video bundle per language. Claude used only as a script QA call (~$0.01–0.05 per video).

Target Languages

English (master) + Spanish, French, German, Portuguese, Arabic, Mandarin, Cantonese, Hindi, Japanese, Nordic (SV/NO/DA/FI) + additional markets per RTT test results.

Tier 1 <100ms produced first per batch run  ·  Tier 2 100–200ms second  ·  Tier 3 200ms+ last

⚠ RTT test results to be supplied by user to configure tier order.

Output Formats

All derived from one master production per video:

  • YouTube — 16:9, 60–90 sec promo
  • Instagram / TikTok Reels — 9:16 vertical short
  • LinkedIn — 1:1 square or 16:9

Video Styles

Mix of three, chosen per use case at job definition time:

  1. AI avatar / talking head — Nexetra persona photo driven by TTS audio (SadTalker / MuseTalk on DGX Spark)
  2. Slides + voiceover — Manim animated text/graphics + narration (Mac Mini 05)
  3. AI visuals + voiceover — ComfyUI B-roll + narration (cinematic/product style, DGX Spark)

└── Export Agent        → Mac Mini 06 (multi-format crop + caption burn-in)

Phases

Phase 0 — Foundation

Install and wire up all services.

  1. Ollama + Qwen2.5 72B Q4 (or Llama3.3 70B Q4) on DGX Spark — the script brain
  2. Ollama + Llama3.1 8B on Mac Minis 01+02 — translation workers
  3. Coqui XTTS-v2 on Mac Minis 03+04 — 17-language local TTS
  4. Kokoro TTS on Mac Minis 03+04 — fast EN voices + extras
  5. Edge-TTS on Mac Mini 06 — free MS API, covers Cantonese + Nordic gaps
  6. ComfyUI on DGX Spark with FLUX (images), CogVideoX/Wan2.1 (video), SadTalker/MuseTalk (talking head)
  7. Whisper.cpp on DGX Spark — local subtitle generation
  8. n8n self-hosted on ubuntu-1 — visual workflow trigger UI
  9. Archon on ubuntu-1 / Mac Mini 05 — YAML deterministic workflow spec
  10. Scaffold nexetra-media/ workspace

Phase 1 — Script + Translation Engine

  1. Build Nexetra script templates — JSON/YAML structures per video type (hook + body + CTA)
  2. Build scriptgen agent — Ollama prompt chain on DGX Spark → structured script
  3. Optional: 1× Claude API call per video for tone/quality polish
  4. Build translate agent — fan-out Ollama calls across Mac Minis 01+02 in parallel
  5. Ingest RTT test results → build language tier list

Phase 2 — Voice Engine

Map each language to a TTS engine:

Language GroupEngineWhere
ES, FR, DE, PT, HI, ZH, JA, ARXTTS-v2 (local)Mac Minis 03+04
EN (primary)Kokoro TTS (local)Mac Minis 03+04
Cantonese, Nordic (SV/NO/DA/FI)Edge-TTS (free MS API)Mac Mini 06
  1. Build tts agent — translated script segment → WAV file per language
  2. FFmpeg post-process — LUFS normalization, silence trimming, sample rate standardization

Phase 3 — Visual Engine (DGX Spark)

  1. Talking head path — SadTalker/MuseTalk drives Nexetra persona photo with TTS audio → MP4
  2. B-roll path — ComfyUI + FLUX generates scene stills from script keywords
  3. Slides path — Manim (Python, 100% local) animated text/graphic explainers on Mac Mini 05
  4. Build Nexetra brand overlay layer — logo watermark, color palette, font overlays

Phase 4 — Assembly + Export

  1. FFmpeg + MoviePy stitches audio + visuals → master 16:9 MP4 with brand watermark
  2. Whisper.cpp generates .SRT per language → burned in or sidecar
  3. Format exporter crops master: 16x9.mp4 (YouTube) · 9x16.mp4 (Reels) · 1x1.mp4 (LinkedIn)
  4. Output path: output/YYYYMMDD-HHMMSS/{video_id}/{lang}/{format}.mp4

Phase 5 — Orchestration

  1. Wire all agents into n8n workflow on ubuntu-1 (webhook or cron trigger)
  2. Mirror as Archon YAML spec — deterministic, CLI-runnable
  3. Per-video job JSON file drives each run (title, languages, style, CTA text)
  4. Run manifests + per-stage logs: run_manifest.json, run_summary.md (PASS/FAIL)

Phase 6 — Language Prioritization

  1. Ingest RTT test result CSV/JSON from user
  2. Build tier list: Tier 1 <100ms Tier 2 100–200ms Tier 3 200ms+
  3. Job queue: Tier 1 languages processed first in every batch run
  4. Output folder tagging by tier for reporting
